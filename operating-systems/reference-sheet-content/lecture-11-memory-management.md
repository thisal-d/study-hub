# LECTURE 11 — Main Memory & Virtual Memory Management

---

## 1. Key Terms

| Term | Definition | Key Point |
|------|-----------|-----------|
| **Logical Address** | Address generated by the CPU (also: virtual address) | What the program sees |
| **Physical Address** | Address seen by the memory unit | Actual RAM location |
| **MMU** | Memory Management Unit — hardware that translates logical → physical | Done at execution time |
| **Base Register** | Holds smallest legal physical address of a process | Lower bound |
| **Limit Register** | Holds size of legal address range | Upper bound protection |
| **Swapping** | Moving processes between RAM and disk to free memory | Increases multiprogramming |
| **Page** | Fixed-size block of logical memory | Divide logical memory |
| **Frame** | Fixed-size block of physical memory | Divide physical memory |
| **Page Table** | Translates page numbers to frame numbers | Per-process; in memory |
| **TLB** | Translation Lookaside Buffer — fast associative memory for page table | Hardware cache |
| **Demand Paging** | Load pages into RAM only when needed (on access) | Lazy loading |
| **Page Fault** | Accessed page is not in RAM | Trap to OS; load from disk |
| **Valid bit** | Page table entry bit: 1=page in memory (legal), 0=page not in memory | Triggers page fault if 0 |

## 2. Address Binding — Three Stages

| Stage | When | Description | Example |
|-------|------|-------------|---------|
| **Compile Time** | At compilation | Absolute addresses embedded in code; must know memory location in advance | MS-DOS .COM programs |
| **Load Time** | When loaded to memory | Compiler generates relocatable code; loader adjusts addresses | Old batch systems |
| **Execution Time** | While running | CPU generates logical addresses → MMU translates to physical | Modern OS (uses Base+Limit or Paging) |

> **Modern OS uses Execution Time binding** (allows process to be moved in memory while running)

---

## 3. Contiguous Allocation

**Each process gets one continuous block of RAM.**

| Aspect | Detail |
|--------|--------|
| Protection | Base register (lower bound) + Limit register (size) |
| Hole allocation strategies | First-fit, Best-fit, Worst-fit |
| **First-fit** | Allocate first hole large enough → fast |
| **Best-fit** | Smallest hole that fits → least waste, but slow search |
| **Worst-fit** | Largest hole → produces largest leftover |
| **Best performers** | First-fit & Best-fit > Worst-fit |

### Fragmentation

| Type | Definition | Solution |
|------|-----------|---------|
| **External Fragmentation** | Total free memory is enough but not contiguous | **Compaction** (shuffle) or **Paging** |
| **Internal Fragmentation** | Allocated block slightly larger than needed; unused space inside partition | Smaller partition sizes |

**Compaction:** Move all processes to one end; merge free holes → only possible with execution-time binding

## 4. Paging (★ EXAM CRITICAL ★)

**Divide physical memory into fixed-size frames; logical memory into same-size pages.**

```
Logical Address:  | page number (p) | page offset (d) |
                  |     m - n bits  |     n bits      |
```
- Page size = 2ⁿ bytes; page number uses (m-n) bits for 2ᵐ total logical space
- Physical address = (frame number from page table) × page size + offset d

### Key Formulas

| Value | Formula |
|-------|---------|
| Number of pages | Virtual memory size / Page size |
| Number of frames | Physical memory size / Page size |
| Offset bits | log₂(page size) |
| Page number bits | Total logical address bits − offset bits |
| Page table entries | = Number of pages (one entry per page) |

### Worked Example (from paper-1, Q2c):
- Physical memory = 256 MB, Virtual memory = 2 GB, Page size = 16 KB
- Pages = 2 GB / 16 KB = 2³¹ / 2¹⁴ = **2¹⁷ = 131,072 pages**
- Frames = 256 MB / 16 KB = 2²⁸ / 2¹⁴ = **2¹⁴ = 16,384 frames**
- Offset bits = log₂(16KB) = **14 bits**
- Page number bits = 31 − 14 = **17 bits**
- Frame number bits = 28 − 14 = **14 bits**
- Page table entries = **131,072** (one per page)

## 5. TLB (Translation Lookaside Buffer)

| Hit | Latency |
|-----|---------|
| TLB hit (found in TLB) | 1 memory access + ε |
| TLB miss (must read page table) | 2 memory accesses + ε |

**EAT = (memory_access + ε) × α + (2×memory_access + ε) × (1-α)**
- α = hit ratio, ε = TLB lookup time

---

## 6. Valid/Invalid Bit in Page Table

| Bit | Meaning |
|-----|---------|
| **Valid (v)** | Page IS in process's logical address space AND is in physical memory |
| **Invalid (i)** | Page is NOT in physical memory (or not in address space) → triggers **page fault** |

---

## 7. Demand Paging & Page Faults (★ EXAM ★)

**Demand Paging:** Pages loaded into RAM only when accessed. Start process with no pages in memory.

### Steps When Page Fault Occurs (★ List all steps ★):
1. Check internal table (PCB) — is reference valid?
2. If invalid reference → terminate process; if valid but not in memory → bring in
3. Find a free frame in physical memory
4. Schedule disk I/O to load the page into the free frame
5. When I/O complete, update page table (set valid bit = 1)
6. Restart the instruction that caused the page fault

### If No Free Frames:
1. Select a **victim frame** using a page replacement algorithm
2. Write victim to disk (if dirty)
3. Load desired page into freed frame
4. Update page table; restart instruction

---

## 8. Page Replacement Algorithms (★ EXAM — calculate page faults ★)

| Algorithm | Description | Anomaly? | Performance |
|-----------|-------------|----------|-------------|
| **FIFO** | Replace oldest page (first loaded) | **Belady's Anomaly** ← more frames can = MORE faults | Simple, poor |
| **OPT (Optimal)** | Replace page not used for longest time in future | None | Best (theoretical; requires future knowledge) |
| **LRU** | Replace page not used for longest time in past | None (stack algorithm) | Good; practical |

### Reference String Worked Example:
**String:** 1,2,3,4,2,1,5,6,2,1,2,3,7,6

**FIFO — 4 frames:** Track: load first 4 unique → fault, then on page 5 replace 1 (oldest) etc.
**LRU — 4 frames:** Replace least-recently-used page on each fault.

> Tip: Track state of frames after each reference. Mark **F** for fault.

## 9. Virtual vs. Physical Address

| Aspect | Logical (Virtual) Address | Physical Address |
|--------|--------------------------|-----------------|
| Generated by | CPU | Memory Unit |
| Seen by | Program/User | Hardware/RAM |
| At compile/load time | Same as physical | Same as logical |
| At execution time | **Different** from physical | Actual RAM location |
| Translation | Via MMU (page table) | — |

---

## 10. Hardware for Virtual Memory

Two hardware resources needed:
1. **Page table** (in RAM — hardware registers or hardware-maintained)
2. **TLB** (Translation Lookaside Buffer — fast associative memory)
