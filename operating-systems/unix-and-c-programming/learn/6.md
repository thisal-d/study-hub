## Introduction
A **thread** is a separate line of execution within a program. Unlike `fork()` which creates a copy of the process, threads run **within the same process** and share the same memory.

**Compilation Warning:**
You **must** link the pthread library when compiling.
> **Command:**
> `gcc program.c -o program -lpthread`

---

## The Thread Function Structure

Any function you want to run as a thread **must** follow this specific signature:

> **Thread Function Signature:**
> ```c
> void* myFunction(void* arg) {
>     // Code here
>     return NULL; // or pthread_exit(NULL);
> }
> ```
*   **Return type `void*`:** It can return a generic pointer (or NULL).
*   **Argument `void* arg`:** It accepts a generic pointer input.

---

## Creating Threads (`pthread_create`)

This system call starts the thread.

> **Syntax:**
> ```c
> pthread_create(&thread_id, NULL, function_name, &argument);
> ```
*   `&thread_id`: Pointer to a `pthread_t` variable to store the thread ID.
*   `NULL`: Default attributes (usually always NULL).
*   `function_name`: The name of the function the thread will run.
*   `&argument`: The address of the data you want to pass to the function.

---

## Waiting for Threads (`pthread_join`)

This is similar to `wait()` for processes. The main program pauses until the thread finishes.

> **Syntax:**
> ```c
> pthread_join(thread_id, NULL);
> ```

## Passing Data to Threads (The `void*` Arg)

This is the most critical part of **[Lab Sheet 5](../labsheets/5.md)**. Since the function only accepts `void*`, you must cast your data.

### Passing simple integers
You pass the address of the integer, then cast it back inside the thread.

> **Example:**
> ```c
> void* printNum(void* arg) {
>     int *ptr = (int*) arg; // Cast void* to int*
>     printf("Number: %d\n", *ptr); // Dereference
>     return NULL;
> }
>
> int main() {
>     int x = 10;
>     pthread_t t1;
>     pthread_create(&t1, NULL, printNum, &x); // Pass address of x
>     pthread_join(t1, NULL);
>     return 0;
> }
> ```

### Passing structures ([Lab 5: Activity 3](../labsheets/5.md#activity-3))
When you need to pass multiple values (like `a` and `b`), define a `struct`.

> **Step 1: Define the struct**
> ```c
> typedef struct {
>     int a;
>     int b;
> } Inputs;
> ```

> **Step 2: Inside `main`, fill the struct and pass its address**
> ```c
> Inputs data;
> data.a = 5;
> data.b = 10;
> pthread_create(&t1, NULL, calculate, &data);
> ```

> **Step 3: Inside the thread, cast back to struct pointer**
> ```c
> void* calculate(void* arg) {
>     Inputs *in = (Inputs*) arg; // Cast void* to Inputs*
>
>     // Access members using ->
>     int sum = in->a + in->b;
>     printf("Sum: %d\n", sum);
>     return NULL;
> }
> ```

> [!TIP]
> **Summary & Quick Recap:**
> - **Threads:** Run tasks in parallel in the same process. Link with `-lpthread`.
> - **Function Signature:** `void* func(void* arg) { return NULL; }`
> - **Creation & Join:** `pthread_create` starts it, `pthread_join` waits for it.
> - **Passing Data:** Use a `struct` to pass multiple values to a thread. Remember to cast `void*` back!

## Practice Questions

### 🌱 Small Coding Questions
1.  **Hello Thread:** Write a program that creates a thread. The thread function should simply print "Thread is running!". The main function should print "Main is running!".
2.  **ID Check:** Modify the above program to pass an integer (e.g., `1`) to the thread. The thread should print "Thread ID: 1".

### 🚀 Larger Coding Question
**Problem:** Write a C program to perform addition and multiplication using separate threads (Based on **[Lab 5: Activity 4](../labsheets/5.md#activity-4)**).

1.  Create a `struct` named `Values` containing two integers `x` and `y`.
2.  In `main`, ask the user to input two numbers. Store them in a `Values` struct instance.
3.  Create **two threads**:
    *   **Thread 1:** Runs a function `addThread`. Pass the struct. It calculates and prints `x + y`.
    *   **Thread 2:** Runs a function `mulThread`. Pass the struct. It calculates and prints `x * y`.
4.  Use `pthread_join` to ensure main waits for both threads.

**Sample Output:**
```text
Enter two numbers: 4 5
Addition Result: 9
Multiplication Result: 20
```
