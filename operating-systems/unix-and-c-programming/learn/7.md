## Introduction
Processes normally don't share memory. If the Parent wants to send data to the Child, they need a channel. A **Pipe** acts like a one-way tunnel: data goes in one end and comes out the other.

**Required Headers:**
```c
#include <unistd.h>   // for pipe, fork, read, write
#include <string.h>   // for strlen
#include <sys/wait.h> // for wait
```

---

## The Pipe Mechanism

### File Descriptors: `fd[0]` and `fd[1]`
When you create a pipe, you pass an array of two integers. These integers are "file descriptors" — they represent the doors to the tunnel.

*   **`fd[0]`**: The **Read End**. (Data comes OUT here).
*   **`fd[1]`**: The **Write End**. (Data goes IN here).

> **How to declare and create a pipe:**
> ```c
> int fd[2];
> if (pipe(fd) == -1) {
>     printf("Pipe failed\n");
>     return 1;
> }
> ```

---

## The Standard Workflow

To communicate between Parent and Child, follow this exact recipe (referenced in **[Lab 6: Activity 1](../labsheets/6.md#activity-1)**):

1.  **Create Pipe:** `pipe(fd)` (Do this **BEFORE** `fork()`).
2.  **Fork:** `fork()` (Now both Parent and Child have a copy of `fd[0]` and `fd[1]`).
3.  **Close Unused Ends:** This is crucial! If you want Parent to write and Child to read:
    *   **Parent:** Closes the read end (`close(fd[0])`). Writes to `fd[1]`.
    *   **Child:** Closes the write end (`close(fd[1])`). Reads from `fd[0]`.

> **Template Code (Parent writes to Child):**
> ```c
> int fd[2];
> pipe(fd);
> pid_t pid = fork();
>
> if (pid > 0) {
>     // --- PARENT PROCESS ---
>     close(fd[0]); // Close read end (Parent is writing)
>     
>     char msg[] = "Hello Child";
>     write(fd[1], msg, strlen(msg) + 1); // +1 to include null terminator
>     close(fd[1]); // Close write end after done
> }
> else {
>     // --- CHILD PROCESS ---
>     close(fd[1]); // Close write end (Child is reading)
>     
>     char buffer[100];
>     read(fd[0], buffer, sizeof(buffer));
>     printf("Child received: %s\n", buffer);
>     close(fd[0]);
> }
> ```

## Writing and Reading Data

Unlike `printf` and `scanf`, pipes use raw `write` and `read` system calls.

*   **`write(fd[1], data, size)`:**
    *   `fd[1]`: Where to write.
    *   `data`: The data (e.g., a string or int variable).
    *   `size`: How many bytes (e.g., `sizeof(int)` or `strlen(str)`).

*   **`read(fd[0], buffer, size)`:**
    *   `fd[0]`: Where to read from.
    *   `buffer`: A variable to store the incoming data.
    *   `size`: Maximum bytes to read.

**Note on Integers:**
You can send integers directly using `sizeof`.
> **Sending an int:**
> ```c
> int num = 100;
> write(fd[1], &num, sizeof(int));
> ```
> **Receiving an int:**
> ```c
> int received_num;
> read(fd[0], &received_num, sizeof(int));
> ```

## Bi-directional Communication (Two Pipes)

In **[Lab 6: Activity 3](../labsheets/6.md#activity-3)**, the Parent sends a number, the Child doubles it, and sends it back. A single pipe is one-way. To do this, you need **two pipes**.

*   **Pipe 1 (`fd1`):** Parent Write -> Child Read.
*   **Pipe 2 (`fd2`):** Child Write -> Parent Read.

**Logic:**
1.  Create `pipe(fd1)` and `pipe(fd2)`.
2.  `fork()`.
3.  **Parent:** Close `fd1[0]` & `fd2[1]`. Write to `fd1[1]`. Read from `fd2[0]`.
4.  **Child:** Close `fd1[1]` & `fd2[0]`. Read from `fd1[0]`. Write to `fd2[1]`.

> [!TIP]
> **Summary & Quick Recap:**
> - **Pipes:** `fd[0]` is for reading (out), `fd[1]` is for writing (in).
> - **The Recipe:** 1. `pipe(fd)`, 2. `fork()`, 3. Close unused ends in both Parent and Child.
> - **I/O:** Use `write()` and `read()`. Can use `sizeof(int)` for numbers.
> - **Two-way:** Use TWO pipes if both Parent and Child need to write and read.

## Practice Questions

### 🌱 Small Coding Questions
1.  **Descriptor Check:** In a Parent-to-Child pipe, which file descriptor does the Parent close? Which one does the Child close?
2.  **Fix the Code:** Identify the error in this snippet:
    ```c
    int fd[2];
    fork(); // Creating process first
    pipe(fd); // Creating pipe second
    // ...
    ```
    *(Hint: Does the child inherit the pipe if it's created after fork?)*

### 🚀 Larger Coding Question
**Problem:** Write a C program where a Child process sends its Process ID (PID) to the Parent process.

1.  Create a pipe.
2.  Create a Child process using `fork()`.
3.  **In the Child:**
    *   Get the PID using `getpid()`.
    *   Write the PID into the pipe.
4.  **In the Parent:**
    *   Read the PID from the pipe.
    *   Print: "Received Child PID: [PID]".

**Sample Output:**
```text
Child: Sending my PID...
Parent: Received Child PID: 12345
```
