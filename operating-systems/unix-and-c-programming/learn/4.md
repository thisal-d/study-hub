## Introduction
A pointer is simply a variable that **stores the memory address** of another variable. Think of it like a treasure map: the variable is the treasure, and the pointer is the map showing exactly where the treasure is buried.

---

## The Basics of Pointers

### How to initialize a pointer
A pointer is declared with the type it points to, followed by an asterisk `*`.

> **How to initialize an int pointer:**
> ```c
> int *pointer_to_int;
> ```

### The `&` (Address of) operator
To make a pointer point to something, you need its address. The `&` symbol means "address of".

> **How to get the address of a variable:**
> ```c
> int number = 10;
> int *ptr;
> ptr = &number; // ptr now holds the address of 'number'
> ```

### The `*` (Dereference) operator
To access or modify the value stored at the address held by the pointer, use `*` again (dereferencing).

> **How to get the value through a pointer:**
> ```c
> int value = *ptr; // value is now 10
> printf("%d", value);
> ```

> **How to change the value through a pointer:**
> ```c
> *ptr = 20; // The original 'number' variable is now 20!
> ```

## Pointers and Functions (Pass by Reference)

In **README 3**, we passed arrays to functions. Now we know that arrays decay into pointers.

Normal variables (like `int`) are passed by **value** (a copy is made). If you want a function to **change** the original variable, you must pass its **address** (pointer).

> **Example: Swapping two numbers**
> ```c
> void swap(int *x, int *y) {
>     int temp = *x;
>     *x = *y;
>     *y = temp;
> }
>
> int main() {
>     int a = 5, b = 10;
>     swap(&a, &b); // Pass addresses
>     // Now a = 10, b = 5
>     return 0;
> }
> ```

## The `void` Pointer (Generic Pointer)

This is essential for **[Lab Sheet 5: Activity 3](../labsheets/5.md#activity-3)**. A `void*` is a pointer that has no specific data type associated with it yet. It is a "generic" pointer.

### Why `void*` is used in Threads
The thread function in POSIX (`pthread_create`) must accept a `void*` argument. This allows you to pass **any type of data** (int, float, struct, array) to the thread.

**Syntax in Thread Function:**
```c
void* myThreadFunction(void* arg) {
    // arg is a void pointer here
}
```

### How to cast a `void*`
You cannot directly do math or access members with a `void*` because the computer doesn't know how many bytes of data it represents. You must **cast** it back to the specific pointer type.

> **Scenario: Passing an `int` to a thread.**
> 1.  **Passing:** You pass the address of the int (which is automatically treated as `void*`).
> 2.  **Receiving:** You cast `void*` back to `int*`.
> 3.  **Using:** You dereference to get the value.

> **Code Example (Simplified from [Lab 5](../labsheets/5.md)):**
> ```c
> void* printNumber(void* arg) {
>     // Step 1: Cast void* to int*
>     int *p = (int*) arg;
>
>     // Step 2: Dereference to get value
>     int value = *p;
>
>     printf("Received value: %d\n", value);
>     return NULL;
> }
> ```

> **Scenario: Passing a `struct` to a thread.**
> This is exactly what happens in [Lab Sheet 5: Activity 3](../labsheets/5.md#activity-3).
> ```c
> typedef struct {
>     int a;
>     int b;
> } Numbers;
>
> void* calculate(void* arg) {
>     // Cast void* to Numbers*
>     Numbers* nums = (Numbers*)arg;
>
>     // Access struct members
>     int sum = nums->a + nums->b;
>     printf("Sum: %d\n", sum);
>     return NULL;
> }
> ```

> **Scenario: Passing an `array` to a thread.**
> Crucial for **[Lab Sheet 5: Activity 4](../labsheets/5.md#activity-4)**. When passing an array, you cast `void* arg` back to `int*`.
> ```c
> void* processArray(void* arg) {
>     // Step 1: Cast void* to int*
>     int* arr = (int*)arg;
>     
>     // Step 2: Use the array just like normal
>     // Note: arr[i] and *(arr + i) are the exact same thing
>     int sum = 0;
>     for(int i = 0; i < 4; i++) {
>         sum += arr[i]; 
>     }
>     
>     printf("Array Total Sum: %d\n", sum);
>     return NULL;
> }
> ```

> [!TIP]
> **Summary & Quick Recap:**
> - **Pointers:** Variables that store memory addresses (`int *ptr`).
> - **`&` and `*`:** `&` gets the address, `*` gets the value at that address (dereferencing).
> - **Pass by Ref:** Let functions change original variables by passing addresses.
> - **`void*` Magic:** Generic pointer. To use it, cast it back first (`(int*)arg`), then use it.

## Practice Questions

### 🌱 Small Coding Questions
1.  **Pointer Basics:** Write a program that declares an integer `num = 100`. Declare a pointer `ptr` that points to `num`. Print the value of `num` using the pointer.
2.  **Void Pointer Casting:** Write a function `void printValue(void* p, char type)`.
    *   If `type` is 'i', cast `p` to `int*` and print the integer.
    *   If `type` is 'f', cast `p` to `float*` and print the float.
    *   *(Hint: Casting syntax is `(int*)p`)*.

### 🚀 Larger Coding Question
**Problem:** Write a program that increments a value using a thread simulation (no actual threads needed yet, just the pointer logic).

1.  Create a function `void increment(void* arg)`.
2.  In `main`, create an integer `count = 10`.
3.  Call `increment` passing the address of `count`.
4.  Inside `increment`:
    *   Cast the `void*` argument to an `int*`.
    *   Increment the value stored at that pointer (`(*ptr)++`).
5.  Back in `main`, print `count`. It should now be 11.

**Expected Output:**
```text
Initial count: 10
Final count: 11
```
