## Introduction
A **process** is a program in execution. In UNIX, creating new processes is done using system calls. Understanding this is vital for the lab test.

**Required Headers:**
```c
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h> // For fork, getpid, execl
#include <sys/wait.h> // For wait
```

---

## Process IDs (PIDs)
Every process has a unique ID.

*   `getpid()`: Returns the ID of the **current** process.
*   `getppid()`: Returns the ID of the **parent** process (the one that created this one).

> **Example:**
> ```c
> printf("My ID: %d\n", getpid());
> printf("My Parent's ID: %d\n", getppid());
> ```

## Creating Processes: `fork()`

This is the most important system call in **[Lab Sheet 4](../labsheets/4.md)**.

### The "Cloning" Concept
When you call `fork()`, the operating system creates an **exact duplicate** of the current process.
*   The **Child** process is an exact copy of the **Parent**.
*   Both processes start executing from the line **right after** `fork()`.

### The Return Value Logic
Since both processes run the same code, how do we tell them apart? `fork()` returns a different value to each!

*   **Returns `0`**: Means "I am the Child process".
*   **Returns `> 0` (PID)**: Means "I am the Parent process" (and the number returned is the Child's ID).
*   **Returns `-1`**: Error (fork failed).

### Separating Parent and Child Code
We use an `if-else` statement to give different tasks to the parent and child.

> **Standard Template (Essential for Exam):**
> ```c
> pid_t pid = fork(); // Create the clone
>
> if (pid == 0) {
>     // Code ONLY for the Child
>     printf("I am the Child. PID: %d\n", getpid());
> } else if (pid > 0) {
>     // Code ONLY for the Parent
>     printf("I am the Parent. PID: %d\n", getpid());
> } else {
>     // Error handling
>     printf("Fork failed.\n");
> }
> ```

## Waiting for Processes: `wait()`

In **[Lab Sheet 4: Activity 3](../labsheets/4.md#activity-3)**, you observe process hierarchies. Usually, the parent should wait for the child to finish before the parent exits.

If the parent finishes first, the child becomes an "Orphan" (adopted by `init` or `systemd`). If the child finishes but the parent doesn't collect its status, it becomes a "Zombie".

> **How to use `wait()`:**
> ```c
> if (pid > 0) {
>     wait(NULL); // Parent pauses here until Child finishes
>     printf("Child has finished.\n");
> }
> ```

## Replacing Processes: `execl()` vs `system()`

In **[Lab 4: Activity 7 & 8](../labsheets/4.md#activity-7)**, you must understand the difference between executing a command with `execl()` vs `system()`.

### The `system()` function
*   Runs a command by spawning a shell, waits for it to finish, and **returns** to your C program.
*   Code written after `system()` **WILL** be executed.
*   Requires: `#include <stdlib.h>`

### The `execl()` system call
*   While `fork()` creates a copy, `execl()` **replaces** the current process memory entirely with a new program.
*   **Key Behavior:** If `execl()` is successful, the code **after** it will **NOT** execute. The original program is gone.
*   Requires: `#include <unistd.h>`

> **Syntax for `execl()`:**
> `execl(path, command_name, arguments..., NULL);`

> **Example: Running the `date` command:**
> ```c
> printf("About to run date...\n");
> execl("/bin/date", "date", NULL);
>
> // This line will NEVER print if execl works
> printf("This will not be printed.\n");
> ```

> [!TIP]
> **Summary & Quick Recap:**
> - **PIDs:** `getpid()` (me), `getppid()` (parent).
> - **`fork()`:** Clones a process. Returns `0` to child, PID to parent. Code after fork runs twice!
> - **`wait()`:** Parent stops and waits for child to prevent Zombies.
> - **`execl()`:** Replaces the entire process. Code after `execl` is destroyed.
>

## Practice Questions

### 🌱 Small Coding Questions
1.  **Double Print:** Look at the code below. How many times will "Hello" print? Why?
    ```c
    printf("Start\n");
    fork();
    printf("Hello\n");
    ```
    *(Hint: `printf` is in the duplicated code)*.

2.  **Exec Check:** Write a program that prints "Start", then uses `execl` to run `ls` (list files: `/bin/ls`), and then prints "End". Run it and see if "End" appears.

### 🚀 Larger Coding Question
**Problem:** Create a C program that behaves like a simple calculator launcher.

1.  Use `fork()` to create a child process.
2.  **In the Child Process:**
    *   Print "Child: Calculating..."
    *   Sleep for 2 seconds (`sleep(2)`).
    *   Print "Child: Done."
3.  **In the Parent Process:**
    *   Wait for the child to finish.
    *   Print "Parent: Child process completed."
4.  **Error Check:** If `fork()` fails, print an error message.

**Sample Output:**
```text
Child: Calculating...
(2 second pause)
Child: Done.
Parent: Child process completed.
```
