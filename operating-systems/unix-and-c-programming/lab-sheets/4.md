**Objectives:** Understand process management system calls and implement process management concepts using C language.

**Process Management System Calls:**

| System call | Function |
| :--- | :--- |
| `fork()` | Create processes in the UNIX system |
| `getpid()` | Get the process ID of the process |
| `getppid()` | Get the parent process ID of the process |
| `exit()` | Exit the process execution |
| `wait()` | Parent process waits until the child processes are terminating |
| `execl()` | Loads and executes a new process, replacing the current one |

## Exit a Process
The `exit()` system call ends the currently running process.
*   `exit(0)`: Normal termination.
*   `exit(non-zero)`: Abnormal termination.

## Process Creation
The `fork()` system call creates a new process (child). It returns:
*   `0` to the child process.
*   Child's PID to the parent process.
*   `-1` if creation fails.

## execl() System Call
The `execl()` system call loads and executes a new program within the same process, replacing the current process image. If successful, the original program does not continue.

### Activity 7
Type the below program and execute. Observe the output and explain the behavior of `execl()`.
```c
#include <stdio.h>
#include <unistd.h>

int main() {
    printf("here comes the date.\n");
    execl("/bin/date", "date", NULL); // null means end of arguments
    printf("that was the date.\n");
    return 0;
}
```
