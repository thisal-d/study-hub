
# 📘 Data Structures & Algorithms (DSA) Study Structure

## 🔹 Part 1: Basic Concepts & Short Questions

### 1. Step Counting

* Analyze the number of steps in a given algorithm.
* Identify time complexity (basic level).

### 2. Binary Tree (Fill in the Blanks)

* Given a partially completed binary tree:

  * Insert 2–4 values into the correct positions.
  * Follow Binary Search Tree (BST) rules if specified.

### 3. Queue Operations

* Perform operations like:

  * `enqueue (push)`
  * `dequeue (pop)`
* Example question:

  * After a sequence of operations, determine:

    * Which value is removed last?
    * Final state of the queue.

### 4. Stack Operations

* Perform operations like:

  * `push`
  * `pop`
* Example question:

  * After a sequence of operations, determine:

    * Which value is popped last?
    * Final state of the stack.

## 🔹 Part 2: Essay / Long Questions

### 1. Binary Tree

#### a. Drawing a Tree

* Construct a binary tree based on a given scenario.

#### b. Insertion

* Add a new node to the tree.
* Show the updated tree.
* Explain the steps clearly.

#### c. Deletion

* Remove a node from the tree.
* Show the updated tree.
* Explain how the tree is adjusted (cases: leaf, one child, two children).

---

### 2. Sorting Algorithms

#### Option A: Merge Sort

* Break array into smaller parts (divide step).
* Merge step-by-step.
* Draw the array after each merge iteration.
* Clearly show how sorted subarrays are formed.

#### Option B: Quick Sort

* Choose a pivot.
* Partition the array around the pivot.
* Show the array after each partition step.
* Continue until fully sorted.
